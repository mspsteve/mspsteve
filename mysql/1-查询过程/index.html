# mysql基础结构

# server层

  包括连接器、查询缓存、分析器、优化器、执行器等，涵盖 MySQL 的大多数核心服务功能，以及所有的内置函数（如日期、时间、数学和加密函数等）， 所有跨存储引擎的功能都在这一层实现，比如存储过程、触发器、视图等

# 存储引擎

  负责数据的存储和提取。其架构模式是插件式的。支持 InnoDB、MyISAM、Memory 等多个存储引擎。 现在最常用的存储引擎是 InnoDB，MySQL 5.5.5 版本开始InnoDB也成为了默认存储引擎。

# 详细介绍

# 连接器

  连接器负责跟客户端建立连接、获取权限、维持和管理连接，使用如下命令创建连接后，如果长时间没有请求进来，则连接器会自动断开，控制参数为wait_timeout控制， 默认时间为8小时(设置连接池时间时需要考虑，避免从连接池拿到的链接为无效链接)。

mysql -h$ip -P$port -u$user -p
1

  建立连接比较耗时，要尽可能地减少建立连接的操作，尽量使用长连接。如果全部使用长连接，会导致mysql占用内存上涨快(连接对象冗余大量其他对象)，一直不断开会导致 内存被干爆导致OOM(mysql重启)，解决方案有两种方式：

  • 定期断开长连接。
  • mysql版本，每次执行比较大的操作，使用mysql_reset_connection来初始化连接资源。

# 查询缓存

  查询语句首先回去内存去查询缓存(上一次查询结果以key-value方式存储，key为语句，value为查询结果)，实际操作性比较差，每次更新都会去清除查询缓存(mysql 8.0没有此功能)。

# 分析器

  词法分析(识别关键字)->语法分析(判断是否满足sql规范)

# 优化器

  • 选择索引
  • 多表级联查询时，决定每个表的连接顺序。

# 执行器

  • 校验是否有权限操作该表
  • 调用存储引擎接口取表的第一个，然后调用存储引擎的下一行，执行判断逻辑
  • 将满足条件的行组织成记录集作为结果集返回
